Manual Sync Preflight CSV 与 Scheduler Sync 流程确认

这张流程图用于确认产品语义：CSV 预览不是 debug mode，不是“按快照执行”的导入文件。 手动流程有两个按钮：Sync Now 直接同步，Preview Changes 才生成 CSV。 真正写入 Snipe-IT 时，手动直接同步、手动预览确认和定时任务都进入同一套实际 sync 入口。

结论先看

不是 Debug

CSV 预览不是一个 debug 按钮

UI 上应该是一个单独的 Preview Changes 按钮。 只有点它才生成 CSV；普通 Sync Now 不生成 CSV。

不按快照

确认后不按第一次 CSV 快照执行

用户确认后，系统重新跑一次 lookup / mapping / planning / import。 如果第二次真实结果和 CSV 有差异，以最终报告为准。

共享入口

真实 sync 只有一套入口

Sync Now、预览确认后的真实执行、scheduler 自动执行都进入同一个实际 sync pipeline。 区别只是触发来源和是否先走 CSV 预览。

整体流程图

Trigger

1Sync Now

用户想立刻执行真实同步。此按钮不生成 CSV，直接进入共享真实 sync pipeline。

1Preview Changes

用户想先看本次可能新增/修改的对象。此按钮只生成临时 CSV 预览。

1Scheduler 到点

后台按 daily / weekly / monthly 计划触发。无人值守，不弹 UI，不生成临时 CSV。

Preflight Gate

2跳过 CSV 预览

Sync Now 是直接执行入口，request 中 ManualPreflightCsvEnabled = false。

2生成临时 CSV 预览

只对 Preview Changes 生效。输出到 C:\ProgramData\AteraSnipeSync\Preflight\{runId}\。

2跳过 CSV 预览

scheduled request 强制设置 ManualPreflightCsvEnabled = false。

Decision

3直接执行

Sync Now 不等待人工审核，不读取 CSV，直接调用真实 sync。

3用户确认或取消

Preview Changes 后，确认才执行真实 sync；取消则停止。确认后不使用 CSV 快照作为执行输入。

3自动继续

scheduler 设置 TriggeredBy = "scheduled"， 然后直接进入真实 sync。

Shared Real Sync

4同一套真实 sync 入口

重新执行 Atera Pull、Reconstruction、Snipe Import planning 和真实写入。 Snipe Import 按 MAC、serial、name similarity 匹配。

5最终报告和状态

真实执行结果单独保存为 final report / status。 它和手动 preflight CSV 文件分开保存。

关键点： Sync Now 直接真实同步；Preview Changes 只生成审核 CSV。 预览确认后会重新计算当前状态并走共享真实 sync pipeline，不按第一次 CSV 快照执行。

手动 Sync 的推荐 UI 语义

UI 元素 推荐命名 行为 不要这样设计
直接同步按钮 Sync Now 直接调用真实 sync pipeline，不生成 preflight CSV，不等待人工确认。 不要把 Sync Now 做成必须先预览的按钮。
预览按钮 Preview Changes 生成三张 CSV，显示 folder path，等待用户确认或取消。 不要叫 Debug Mode，避免误解成全局调试开关。
确认按钮 Confirm and Sync 重新跑一次真实 sync，不读取第一次 CSV 作为执行输入。 不要叫 Import This CSV，因为并不是按 CSV 快照导入。
取消按钮 Cancel 停止流程，不执行任何真实 Snipe-IT POST/PATCH。 不要保存成 pending job，第一版没有快照强一致。
CSV 文件夹 Preflight Folder 显示 C:\ProgramData\AteraSnipeSync\Preflight\{runId}\ 并提供打开入口。 不要和最终报告放在同一个位置或同一种 report 概念里。

真实 Sync Pipeline

1. Atera Pull

从 Atera 拉 agent/device inventory。自动测试只用 mock，不调用真实 API。

2. Reconstruction

把 Atera device 转成 SnipeImportBatch，包含 serial、MAC、company、model、notes。

3. Snipe Import Planning

查 company、category、manufacturer、model、hardware，并按 MAC、serial、name similarity 决定 add/modify。

4. Snipe-IT Writes

真实执行时发送 company/model/hardware 的 POST 或 PATCH。scheduled sync 不生成 preflight CSV。

5. Final Report

保存真实 created/updated/failed/warnings/actions。手动 preflight CSV 不等于最终报告。

6. Status / Notification

WorkerService 后续会保存最新状态并发通知。scheduler 不等待人工确认。

实现约束对应关系

需求 当前实现点 含义
Preview Changes 先写 CSV SnipeImportOptions.ManualPreflightCsvEnabled + SnipeImportPreflightCsvWriter 只有预览流程打开该选项。Importer 能在真实写入前输出三张计划表；CSV 写失败会阻止 POST/PATCH。
Sync Now 直接执行 手动真实 sync request Sync Now 应设置 ManualPreflightCsvEnabled = false，直接进入共享真实 sync pipeline。
scheduler 不生成 CSV ScheduledSyncRequestFactory.CreateScheduledRequest scheduled request 会强制关闭 ManualPreflightCsvEnabled。
真实 sync 共享入口 后续 SyncOrchestrator / WorkerService wiring 手动确认后和定时任务都应调用同一个 orchestration pipeline，只是 request metadata 不同。
不按快照执行 产品流程约束 确认后重新计算。CSV 仅供审核，不作为第二次执行的输入文件。
