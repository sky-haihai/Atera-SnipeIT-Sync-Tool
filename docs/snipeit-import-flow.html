比较优先级

Priority 1

MAC address 精确匹配

导入前先规范化 MAC 地址，再查找 Snipe-IT 中是否存在同一硬件身份。只允许唯一命中自动更新。

Priority 2

Serial number 精确匹配

MAC 未命中时使用 serial number。serial 仍是强身份，但不再是第一优先级。

Priority 3

Asset name 高相似度匹配

MAC 和 serial 都未命中时，才使用名称相似度。必须是单一、高置信度候选，否则跳过或记录待人工确认。

比较规则矩阵

顺序 比较键 规范化建议 自动更新条件 失败/歧义处理
1 MAC address 去分隔符、去空白、统一大小写；忽略空值和明显无效值。 至少一个导入 MAC 与一个 Snipe-IT asset 唯一匹配。 多个 asset 命中同一 MAC 时，不更新，记录 ambiguous match failure。
2 Serial number trim、统一大小写策略；空白、占位值或未知值不参与匹配。 MAC 无唯一命中，serial 与一个 Snipe-IT asset 唯一匹配。 多个 serial 命中时，不继续按名称覆盖，记录 duplicate serial failure。
3 Asset name trim、压缩空白、大小写归一、可忽略常见后缀；具体算法在技术规格中定。 MAC/serial 均无命中，且只有一个名称候选达到“很高相似度”。 没有高置信度候选则创建新 asset；多个高分候选则跳过并记录待人工确认。

实现状态： Snipe Import 技术规格和代码已采用“MAC 优先、serial 第二、名称高相似度第三”。早期总策划案中 “serial primary identity”的描述已被本模块规格覆盖；后续如维护总策划案，可同步更新该概述。

导入主流程

1. 接收 batch

读取 SnipeImportBatch、import options、dry-run 标志。

2. 校验记录

检查 name、company、model、status、MAC/serial 等匹配输入。

3. 确保依赖

ensure company，ensure model；dry-run 只生成计划动作。

4. 比较 asset

按 MAC、serial、名称高相似度顺序找现有 asset。

5. 决策动作

唯一命中则 update；无命中则 create；歧义则 skip/fail。

6. 汇总结果

返回 created、updated、skipped、failed、warnings、actions。

单条 Asset 决策树

Match path

查找顺序

  • 如果存在有效 MAC，优先查 MAC。
  • 如果 MAC 唯一命中，计划 update，不再继续 serial/name 匹配。
  • 如果 MAC 未命中，查 serial。
  • 如果 serial 唯一命中，计划 update。
  • 如果 MAC/serial 都未命中，执行名称高相似度比较。
  • 如果名称只有一个高置信度候选，计划 update。
Create or stop

无命中与歧义

  • 没有任何候选时，计划 create hardware asset。
  • 多个 MAC 命中、多个 serial 命中，或多个名称高分候选时，不自动更新。
  • 依赖 company/model 缺失且不允许创建时，跳过该 asset。
  • dry-run 下所有写入动作都只记录 WasExecuted = false。

导入动作语义

场景 动作 结果计数 备注
MAC 唯一命中 Update existing asset UpdatedAssets += 1 最高置信度路径。
MAC 未命中，serial 唯一命中 Update existing asset UpdatedAssets += 1 第二层强身份路径。
MAC/serial 未命中，名称单一高相似候选 Update existing asset UpdatedAssets += 1 低于强身份匹配，建议在 action message 中标注 match reason。
没有匹配到 asset Create hardware asset CreatedAssets += 1 需要 company/model/status 等依赖已经可用。
匹配结果歧义 Skip or fail item SkippedAssets 或 FailedAssets 建议作为 failure，因为自动更新可能覆盖错误资产。
Snipe-IT 返回 HTTP 200 但 JSON body 为 error Fail item FailedAssets += 1 必须读取 response body 的业务状态，不能只看 HTTP status。
